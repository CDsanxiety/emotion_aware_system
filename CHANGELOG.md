# 暖暖 (Nuannuan) V2.0 Pro - 修改日志

## 日期：2026-04-18

## 概述

本次修改实现了从"脉冲式响应"到"自主感知唤醒"的重大升级，彻底释放了 `app.py` 中预留的 `agent_loop`，并完善了 `blackboard.py` 的变化检测机制。同时新增了**多代理协作决策系统 (Agentic Reasoning)** 和**硬件闭环 PAD 物理表达**，实现"老友式"智能回馈和"动起来都有性格"的物理表达。

## 第二部分：决策升级：多代理协作 (Agentic Reasoning)

### 1. 核心功能

实现三个专业代理分工协作：

| 代理 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **感知代理** | 解析 VLM 细节 | vision_desc, audio_text | 提取的细节列表、场景上下文、潜在问题 |
| **记忆代理** | 检索历史上下文 | 当前上下文、感知结果 | 相关记忆、用户画像、健康状态、特殊日期 |
| **执行代理** | 综合决策生成 | 感知结果、记忆结果 | 最终回复、情绪标签、动作、音乐类型 |

**核心特性**：
- **抑制规则**：避免增加用户压力（如主人生病时不提醒洗杯子）
- **场景感知**：根据用户状态调整回复策略
- **记忆增强**：结合历史记录生成个性化回复

### 2. 技术实现

#### 2.1 multi_agent.py (新增)

**PerceptionAgent (感知代理)**
- 提取关键物体细节（杯子、桌子、环境等）
- 分析场景类型（回家、客厅、工作、休息等）
- 检测潜在关注点（主人累了、光线不足等）

**MemoryAgent (记忆代理)**
- 检索长期记忆相关上下文
- 提取用户画像（工作繁忙、感冒中、心情不好等）
- 检查特殊日期（生日、节日等）
- 评估用户健康状态

**ExecutionAgent (执行代理)**
- 应用抑制规则（避免增加压力）
- 生成个性化回复
- 选择合适的情绪和动作
- 提供音乐类型建议

**MultiAgentCoordinator (协调器)**
- 协调三个代理的协作流程
- 执行完整的 Agentic Reasoning 流程

#### 2.2 llm_api.py 集成
- 新增多代理协调器单例
- 新增 `get_response_with_multi_agent()` 函数
- 扩展返回结果，包含推理链和抑制规则

#### 2.3 app.py 启用
- 修改导入：`from llm_api import get_response_with_multi_agent as get_response`

### 3. 完整推理流程示例

#### 场景：主人昨天感冒了，今天桌上有个没洗的杯子

```
输入:
  vision_desc = "主人坐在客厅，桌上有个没洗的杯子，看起来有点累"
  audio_text = "（暂无语音）"
  emotion = "neutral"

感知代理分析:
  - 提取细节: [人物]主人, [杯子]杯子, [表情]累, [环境]乱
  - 场景上下文: 客厅放松
  - 潜在问题: 主人看起来累, 未洗的杯子

记忆代理检索:
  - 检索记忆: "主人昨天感冒了"
  - 用户画像: {"感冒中": True}
  - 健康状态: 需要关心

执行代理决策:
  - 应用抑制规则: ["不要提醒洗杯子（会增加压力）"]
  - 推理链:
    1. 原始情绪识别: neutral
    2. 感知细节: [人物]主人, [杯子]杯子, [表情]累, [环境]乱
    3. 相关记忆: 主人昨天感冒了
    4. 用户健康状态: 需要关心
    5. 用户画像: ['感冒中']
    6. 触发抑制规则: 不要提醒洗杯子（会增加压力）

  - 最终回复: "我注意到你看起来有点累，身体还好吗？如果不舒服一定要好好休息哦，需要我放一些舒缓的音乐帮你放松吗？"
  - 情绪标签: caring
  - 动作: comfort
  - 音乐类型: music_relax
```

## 第三部分：硬件闭环：PAD 映射到"肢体语言"

### 1. 核心功能

将 PAD 三维情绪模型映射到机器人物理动作，实现"不仅说话有感情，动起来都有性格"。

### 2. 技术实现

#### 2.1 physical_expression.py (新增)
- **PADToPhysicalMapper**：将 PAD 值映射到物理表达参数
- **PhysicalExpressionController**：执行具体的物理动作，发布到 ROS
- 支持 LED 灯带颜色/亮度/动画
- 支持头部动作速度/角度/点头频率
- 支持肢体语言类型和面部表情

#### 2.2 pad_model.py 扩展
- 新增 `bind_expression_controller()` 方法
- 新增 `get_physical_expression()` 方法
- 新增 `get_pad_values()` 方法
- 新增 `get_emotion_state_name()` 方法
- 扩展情绪类型支持（caring, supportive, empathetic, helpful）

#### 2.3 ros_client.py 扩展
- 新增 `publish_physical_expression()` 方法
- 支持物理表达消息发布

#### 2.4 llm_api.py 集成
- 初始化物理表达控制器
- 绑定到 PAD 引擎
- 输出 PAD 和物理表达日志

### 3. PAD 到物理表达的映射规则

#### 3.1 LED 灯带映射 (基于 P 愉悦度 + A 激活度)

| PAD 状态 | 颜色 | 亮度 | 动画 |
|---------|------|------|------|
| P高 + A高 (开心兴奋) | 暖白色 | 80% | pulse (脉动) |
| P高 + A低 (放松愉悦) | 柔粉色 | 60% | steady (稳定) |
| P低 + A高 (紧张不安) | 警示橙 | 70% | pulse (脉动) |
| P低 + A低 (悲伤疲惫) | 忧郁紫 | 40% | breathe (呼吸) |
| D高 (自信) | 能量绿 | 70% | pulse (脉动) |
| D低 (顺从关怀) | 温和暖色 | 50% | steady (稳定) |

#### 3.2 头部动作映射

| 维度 | 高值效果 | 低值效果 |
|------|---------|---------|
| **P (愉悦)** | 头部上扬，轻快动作 | 头部下垂 |
| **A (激活)** | 转头速度快，点头频繁 | 转头速度慢，静止 |
| **D (优势)** | 头部挺直，自信 | 头部微微下垂，顺从 |

#### 3.3 肢体语言类型

| 条件 | 肢体语言 |
|------|---------|
| P高 + D高 | welcoming (欢迎) |
| P高 | happy (开心) |
| D低 | caring (关怀) |
| A高 | alert (警觉) |
| P低 | subdued (低沉) |

### 4. ROS 消息格式

```json
{
  "type": "physical_expression",
  "data": {
    "led": {
      "color": [255, 200, 150],
      "brightness": 0.8,
      "animation": "pulse"
    },
    "head": {
      "yaw_angle": -10.0,
      "pitch_angle": 5.0,
      "speed": 1.5,
      "nod_frequency": 1.5
    },
    "body_language": "caring",
    "facial_expression": "concerned"
  }
}
```

### 5. 日志输出示例

```

[物理表达] LED: 暖白 80% | 动画: 脉动 | 头部: yaw=-0.0° pitch=-5.0° 速度=1.5 | 肢体: 欢迎
```

## 依赖修复历史

### CogVLM 导入问题
- **问题**：NumPy 2.x 与 PyTorch 不兼容，导致 torch 导入失败
- **解决**：在 `_agent_worker()` 内部延迟导入，失败时设置 `COGVLM_AVAILABLE = False`

### FER 类导入问题
- **问题**：fer 模块的 `__init__.py` 未导出 FER 类
- **解决**：从 `fer.fer` 直接导入
